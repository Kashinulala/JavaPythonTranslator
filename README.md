# JavaPythonTranslator
---
### Краткое описание:
Транслятор состоит из четырех основных частей:
1. Лесический анализатор
2. Синтаксический анализатор
3. Семантический анализатор
4. Кодогенератор

Лесический анализатор и синтаксический анализатор реализованны с помощью ANTLR с использованием грамматики Java описанной в [официальной документацие языка](https://docs.oracle.com/javase/specs/jls/se7/html/jls-18.html).
Семантический анализатор и кодогенератор унаследованны от стандартного класса ANTLR - Visitor. 
Ознакомится с кодом транслятора можно в TranslatorLibrary.

### Возможности:
- Лексический анализ Java кода с поддержкой основных конструкций
- Синтаксический анализ и построение абстрактного синтаксического дерева (AST)
- Генерация Python кода на основе AST
- Графический интерфейс реализованные как веб-страница
- Поддержка основных конструкций Java:
  - Переменные и массивы
  - Условные операторы (if/else/switch...case)
  - Циклы (for, while, do-while)
  - Консольный вывод
  - Арифметические и логические операции
  - Метод main() и класс в котором он находится

### Структура проекта
Транслятор реализован по принципу клиент-серверной архитектуры.
Точкой входа можно считать файл JavaPythonTranslator/Program.cs который отвечает за запуск backend'а.
Транслятор реализованный как библиотека классов. Файлы транслятора находятся в TranslatorLibrary. Он используется backend'ом.
Директория frontend хранит в себе файлы веб-страницы. app.js обрабатывает всю логику frontend'а.

### Инструкция по запуску:
1. Склонировать репозиторий.
2. Запустить backend (asp.net проект) используя: 
```
dotnet run --urls "https://localhost:7240; http://localhost:5135"
```
3. Запустить frontend используя, например, NodeJS:
```
npm install -g http-server 
```
```
http-server -p 8080
```
4. Веб-страница доступна по пути: http://127.0.0.1:8080/

Если вы запустили frontend на другом адресе/порту необходимо перенастроить CORS в .net проете. Это делается в файле JavaPythonTranslator/Program.cs. Затем перезапустить backend.
